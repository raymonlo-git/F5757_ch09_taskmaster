# Commit message rules
Following the following rules to compose the messages

- New function: `feat(<scope>): <a dscription>`
- Fix: `fix(<scope>): <a dscription>`
- Refactor: `refactor(<scope>): <a dscription>`

e.g.
- `feat(gui): A new review page is added before confirmation`
- `fix(api): Fixed a 404 error`
- `refactor(db): Refactored the stored-proc to make it more efficient`
